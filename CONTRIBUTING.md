# CONTRIBUTING.md

## 開発フロー

- main ブランチは常にデプロイ可能
- develop ブランチから feature/xx ブランチを切って作業
- develop ブランチにPRを出し、レビュー後にマージ

## ブランチルール

- feature/xxx 機能追加
- fix/xxx バグ修正
- docs/xxx ドキュメント更新 など

## コミットメッセージ

- 1行目にプレフィックスをつける:
  - `feat`: 機能追加・修正
  - `fix`: バグフィックス
  - `docs`: ドキュメント追加・修正
  - `style`: CSS スタイルのみの修正
  - `refactor`: リファクタによる修正
  - `test`: テストのみの修正
  - `chore`: そのほか。上に当てはまらないもの。
- 1行目: 要約：例「投稿フォームの作成をおこなった」
- 2行目: 空行
- 3行目以降: 補足を箇条書きで

## PR作成手順

- タイトルは日本語で明確に
- テンプレートを使用
- CIが通っていることを確認
