# 🚀 プロダクト MVP (Minimum Viable Product) 定義ドキュメント

## 1. プロダクト概要と MVP 目標

| 項目             | 定義                                                                                                                                        |
| :--------------- | :------------------------------------------------------------------------------------------------------------------------------------------ |
| **プロダクト名** | （仮称）シンプル・シフト・AI アシスタント                                                                                                   |
| **対象顧客**     | 20 名程度のヘルパーを抱える在宅介護事業所の管理者                                                                                           |
| **MVP のゴール** | **「予定のすっぽかし」と「変更時の調整負担」を解消**し、現場の管理者が上司を説得できるだけの**明確な作業効率化**を達成する。                |
| **差別化要素**   | 既存の高機能システムから切り離された、**AI による変更調整の容易性**に特化。無料プランでコアな価値を提供し、現場からの草の根導入を促進する。 |

---

## 2. MVP の核となる機能要件

MVP では、以下の 3 つの核となる要件に絞り込みます。

### 📝 要件 1：予定の一元化と見える化（担当者ごと、利用者ごと）

| 機能名                           | 詳細                                                                                                                                                           | 達成目標（ユーザー価値）                                                                                                   | 制限事項（MVP）                                                            |
| :------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------- |
| **基本スケジュール登録**         | 利用者名、サービス内容、曜日・時間、担当ヘルパーの情報を一括で登録できるシンプルな画面。CSV/スプレッドシートからのインポート機能は最低限のフォーマットで対応。 | 半年単位の「基本の予定」をシステム上に再現し、**変更前の状態を事務所側で一元管理**する。                                   | 勤怠記録や給与計算への連携は含まない。複雑な繰り返し設定は最低限に留める。 |
| **リアルタイム・マイカレンダー** | ヘルパー（担当者）がスマートフォンでログインし、**自身の最新の確定シフトのみ**を週間または月間カレンダー形式で閲覧できる。                                     | ヘルパー自身による**スケジュールすっぽかしの防止**。ヘルパーは事務所に確認しなくても、どこでも自分の最新情報を把握できる。 | ヘルパー側からのシフト変更申請機能は含まない（変更は事務所側のみが行う）。 |
| **ビュー切り替え**               | 事務所側の管理者画面で、予定を「**担当者（ヘルパー）ごと**」と「**利用者ごと**」で即座に切り替え表示できる。                                                   | スケジュールの全体像と、特定の利用者・ヘルパーに紐づく予定を容易に確認できる。                                             |                                                                            |

---

### 🛠️ 要件 2：週間・月間予定の作成と変更の容易性

| 機能名               | 詳細                                                                                                               | 達成目標（ユーザー価値）                                                             | 制限事項（MVP）                                            |
| :------------------- | :----------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------- | :--------------------------------------------------------- |
| **変更登録（手動）** | 事務所側で、既存の予定をマウス操作（ドラッグ＆ドロップ）で「時間変更」「担当者変更」「キャンセル」を容易に行える。 | 事務所側が利用者やヘルパーの都合で発生した**イレギュラーな変更を迅速に反映**できる。 |                                                            |
| **変更の自動通知**   | 事務所が変更をシステム上で確定した際、**影響を受けるヘルパーのみ**に自動でプッシュ通知が送られる。                 | ヘルパーへの個別連絡の手間をゼロにする。                                             | 事務所側からヘルパーへ個別のメッセージ送信機能は含まない。 |

---

### 🧠 要件 3：AI による変更調整の指示（差別化要素）

| 機能名                        | 詳細                                                                                                                                                      | 達成目標（ユーザー価値）                                                                                 | 制限事項（MVP）                                                                                                              |
| :---------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------- |
| **AI 代行者シミュレーション** | 欠員（キャンセル）が発生した際、事務所側がその枠を選択すると、AI が以下の条件を満たす**代替ヘルパーの候補リスト**を瞬時に表示する。                       | ヘルパーが業務時間外に**代行調整を行う負担を解消**する。複雑な玉突き調整を事務所側で短時間で完了できる。 | **スキル設定の簡略化**：利用者とヘルパーの紐付けに基づき、代行者候補を絞り込む。負荷平準化のロジックは初期的なものに留める。 |
|                               | - 担当可能なスキル/資格を持つ者                                                                                                                           |                                                                                                          |                                                                                                                              |
|                               | - その時間帯に空きがある者                                                                                                                                |                                                                                                          |                                                                                                                              |
|                               | - **負荷が集中していない者**（基本ロジック）                                                                                                              |                                                                                                          |                                                                                                                              |
| **負荷バランサー警告**        | AI が、変更後のスケジュールを基に**各ヘルパーの月間総担当時間**を分析し、事前に設定された「平均負荷」から大きく逸脱する場合（例：±20%）に警告表示を行う。 | 管理者が**業務の平準化**に配慮しながら変更を確定できるようサポートする。                                 | 警告表示のみ（自動修正機能は含まない）。                                                                                     |
| **利用制限**                  | **フリープラン**の場合、AI 代行者シミュレーションの利用は**月 5 回まで**、ヘルパー登録は**5 名まで**とする。                                              | 現場管理者にコアな AI 機能の価値を無料で体験させ、有料プランへの移行を促す。                             |                                                                                                                              |

---

## 3. 非搭載機能（Out of Scope for MVP）

以下の機能は、MVP には含まず、ユーザーからのフィードバックや収益化が進んだ段階で検討します。

- 勤怠管理機能（打刻、休憩記録、残業計算など）
- 給与計算機能
- ヘルパー側からのシフト変更申請やメッセージング機能
- 詳細なスキル・資格管理（複雑な検索条件、有効期限管理など）
- 詳細な権限管理（管理者ロールの多層化など）
- 外部サービス（カイポケ等）との API 連携
