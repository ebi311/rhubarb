## 🔒 プロダクト 非機能要件 定義ドキュメント (ISO 25010 準拠)

### 1. 適合性 (Suitability)

| 品質特性       | 要件項目                 | 要件レベル (MVP)                                                                                                                         |
| :------------- | :----------------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **機能適合性** | 介護業界の慣習への適合性 | 訪問介護事業所の**「利用者ごとの固定スケジュール管理」**と**「イレギュラーな変更対応」**というコアな業務フローに完全に適合していること。 |
| **正確性**     | スケジュール情報の正確性 | **管理者画面とヘルパーのマイカレンダーの情報がリアルタイムで完全に一致**していること。情報の不一致が許容されないこと。                   |

---

### 2. 性能効率性 (Performance Efficiency)

| 品質特性           | 要件項目                            | 要件レベル (MVP)                                                                                                |
| :----------------- | :---------------------------------- | :-------------------------------------------------------------------------------------------------------------- |
| **時間効率性**     | 応答時間                            | 管理者によるスケジュール変更の登録、およびヘルパーによる**マイカレンダー表示は 3 秒以内**に完了すること。       |
|                    | AI 代行者シミュレーションの応答時間 | 管理者が欠員枠を選択してから、**AI による代行候補リストの提示は 10 秒以内**に完了すること（業務効率化のため）。 |
| **リソース利用率** | サーバ負荷                          | **同時接続数 50 ユーザ**（管理者数名＋ヘルパー全員）がピーク時にアクセスしても、上記応答時間を維持できること。  |

---

### 3. 使用性 (Usability)

| 品質特性          | 要件項目     | 要件レベル (MVP)                                                                                                       |
| :---------------- | :----------- | :--------------------------------------------------------------------------------------------------------------------- |
| **理解性/習得性** | 学習コスト   | 管理者が**操作マニュアルなしで 30 分以内**に基本スケジュールを登録し、変更作業を完了できること。                       |
|                   | UI/UX        | ヘルパー（高齢者も含む）がスマートフォンで**マイカレンダーを直感的に閲覧**でき、迷わないシンプルな画面構成であること。 |
| **操作性**        | 変更の容易さ | 管理者画面での**ドラッグ＆ドロップによるシフトの変更操作**がスムーズに行えること。                                     |

---

### 4. 信頼性 (Reliability)

| 品質特性       | 要件項目         | 要件レベル (MVP)                                                                                                                                   |
| :------------- | :--------------- | :------------------------------------------------------------------------------------------------------------------------------------------------- |
| **成熟性**     | サービス稼働時間 | **サービス稼働率は月間 99.9%以上**とすること（月間の停止時間は最大 43 分 20 秒）。特にシフトが確定・変更される時間帯（早朝・夜間）の安定性を確保。 |
| **障害許容性** | データ保全       | **データセンター障害が発生した場合でも、データ損失がゼロ**であること。直近のバックアップからの復旧が可能であること。                               |
| **回復性**     | サービス復旧     | 障害発生後、**サービスが 30 分以内に復旧**できること。                                                                                             |

---

### 5. 保守性 (Maintainability)

| 品質特性         | 要件項目     | 要件レベル (MVP)                                                                                                                                                  |
| :--------------- | :----------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **モジュール性** | 機能分離     | 勤怠・給与計算機能は MVP では含めないため、将来的な**機能拡張時に容易に組み込めるモジュール設計**とすること。特に AI シミュレーションロジックは分離しておくこと。 |
| **変更容易性**   | アップデート | 軽微なバグ修正や UI 改善は、**ユーザへの影響を最小限**に抑え、営業時間外（深夜など）に実施できること。                                                            |

---

### 6. セキュリティ (Security)

| 品質特性     | 要件項目     | 要件レベル (MVP)                                                                                                                                                    |
| :----------- | :----------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **機密性**   | 個人情報保護 | **利用者名、ヘルパー名**を含む全てのデータは、通信時（SSL/TLS）およびデータベース内（暗号化）で保護されていること。                                                 |
| **認証**     | アクセス制御 | ヘルパーは**自身の ID/パスワード（または一意の認証コード）**でのみログイン可能とし、他者のスケジュールを閲覧できないこと。管理者は多要素認証（MFA）を推奨すること。 |
| **否認防止** | ログ記録     | 管理者による**スケジュール変更（誰が、いつ、何を）の履歴をすべて記録**し、遡及して確認できること。                                                                  |

---

### 7. 移植性 (Portability)

| 品質特性          | 要件項目     | 要件レベル (MVP)                                                                                                                 |
| :---------------- | :----------- | :------------------------------------------------------------------------------------------------------------------------------- |
| **共存性/導入性** | デバイス対応 | ヘルパーのマイカレンダー機能は、**主要なスマートフォン OS（iOS, Android）の最新バージョンおよび N-1 世代**で適切に動作すること。 |
