## 3. Interfaces

### 3.1 Proposal Schema（`src/models/aiChatMutationProposal.ts`）

```typescript
// UC1: 担当変更
const ChangeShiftStaffProposalSchema = z.object({
	type: z.literal('change_shift_staff'),
	shiftId: z.string().uuid(),
	toStaffId: z.string().uuid(),
	toStaffName: z.string().optional(), // AI が表示名で返す場合のみ
	reason: z.string().max(200).optional(),
});

// UC2: 時間変更
const UpdateShiftTimeProposalSchema = z.object({
	type: z.literal('update_shift_time'),
	shiftId: z.string().uuid(),
	startAt: z.string().datetime(), // ISO8601 UTC
	endAt: z.string().datetime(),
	reason: z.string().max(200).optional(),
});

export const AiChatMutationProposalSchema = z.discriminatedUnion('type', [
	ChangeShiftStaffProposalSchema,
	UpdateShiftTimeProposalSchema,
]);
export type AiChatMutationProposal = z.infer<
	typeof AiChatMutationProposalSchema
>;

// Execute input（Phase 3 で追記）
export const ExecuteAiChatMutationInputSchema = z.object({
	proposal: AiChatMutationProposalSchema,
	allowlist: z.object({
		shiftIds: z.array(z.string().uuid()).min(1).max(10),
		staffIds: z.array(z.string().uuid()).max(20).optional(),
	}),
	chatSessionId: z.string().optional(), // 監査ログ紐付け用
});
export type ExecuteAiChatMutationInput = z.infer<
	typeof ExecuteAiChatMutationInputSchema
>;
```

### 3.2 allowlist ガードのルール

| チェック項目                                 | 実施場所                                        | 対象              |
| -------------------------------------------- | ----------------------------------------------- | ----------------- |
| `shiftId` が allowlist 内か                  | UI（parseProposal.ts）+ Server（executeAction） | UC1/UC2           |
| `toStaffId` が allowlist 内か                | UI + Server                                     | UC1               |
| office 境界（shift の client が同一 office） | Server のみ                                     | UC1/UC2           |
| シフトステータス（canceled/completed 不可）  | Server のみ                                     | UC1/UC2           |
| 過去シフト不可                               | Server のみ                                     | UC1/UC2           |
| 時刻妥当性（startAt < endAt）                | Schema（Zod）                                   | UC2               |
| スタッフ競合（重複時間帯）                   | Server のみ                                     | UC1/UC2           |
| クライアント競合（重複時間帯）               | Server のみ                                     | UC2（時間変更時） |

## 4. Data Model: `ai_operation_logs`

```sql
-- supabase/migrations/20260317000000_create_ai_operation_logs.sql

CREATE TABLE public.ai_operation_logs (
  id              uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  office_id       uuid        NOT NULL,                          -- 事業所ID（RLS用）
  actor_user_id   uuid        NOT NULL,                          -- 実行者（auth.uid()）
  source          text        NOT NULL CHECK (source = 'ai_chat'),
  operation_type  text        NOT NULL,                          -- 'change_shift_staff' | 'update_shift_time'
  targets         jsonb       NOT NULL,                          -- { shiftId, toStaffId? }
  proposal        jsonb       NULL,                              -- 提案全文
  request         jsonb       NULL,                              -- execute input（allowlist含む）
  result          jsonb       NULL,                              -- { success, errorCode?, conflictShiftIds? }
  created_at      timestamptz NOT NULL DEFAULT now()
);

COMMENT ON TABLE public.ai_operation_logs IS 'AIチャット経由のDB更新操作の監査ログ';

-- インデックス
CREATE INDEX ai_operation_logs_office_id_idx ON public.ai_operation_logs (office_id);
CREATE INDEX ai_operation_logs_created_at_idx ON public.ai_operation_logs (created_at DESC);

-- RLS
ALTER TABLE public.ai_operation_logs ENABLE ROW LEVEL SECURITY;

-- 同一 office_id の admin のみ閲覧可
CREATE POLICY "Admins can read their office logs"
ON public.ai_operation_logs
FOR SELECT
TO authenticated
USING (
  EXISTS (
    SELECT 1 FROM public.staffs s
    WHERE s.auth_user_id = auth.uid()
      AND s.office_id = ai_operation_logs.office_id
      AND s.role = 'admin'
  )
);

-- INSERT は service_role（サーバサイド）のみ
-- → Server Action は createAdminClient() で書き込む OR RLS 無効クライアントを使用
-- （既存の admin.ts の createAdminClient() パターンを踏襲）
```

### RLS 方針まとめ

| 操作          | 主体                          | 手段                                          |
| ------------- | ----------------------------- | --------------------------------------------- |
| SELECT        | 同一 office の admin          | RLS `USING` ポリシー                          |
| INSERT        | Server Action（service_role） | `createAdminClient()` で RLS バイパス書き込み |
| UPDATE/DELETE | 禁止（監査ログは不変）        | ポリシー未定義 = DENY                         |

## 5. Service 競合チェック設計（Issue #121 再利用候補）

### UC1: change_shift_staff

```
1. getAdminStaff(userId)           → 認可 + officeId 取得
2. shiftRepo.findById(shiftId)     → 存在確認
3. status 確認（canceled/completed → 400）
4. 過去シフト確認（start_time < now() → 400）
5. staffRepo.findById(toStaffId)   → 存在 + 同一 office
6. shiftRepo.findStaffConflictingShifts(
     toStaffId, startTime, endTime, officeId, excludeShiftId=shiftId
   )                               → 競合あり → 409 ConflictError
7. shiftRepo.updateStaffAssignment(shiftId, toStaffId, notes?)
8. auditLogRepo.insert(...)        → success ログ
```

### UC2: update_shift_time

```
1. getAdminStaff(userId)           → 認可 + officeId 取得
2. shiftRepo.findById(shiftId)     → 存在確認 + clientId / staffId 取得
3. status 確認
4. 過去シフト確認
5. 時刻妥当性（startAt < endAt）
6. shiftRepo.findClientConflictingShifts(
     clientId, newStart, newEnd, officeId, excludeShiftId=shiftId
   )                               → 競合あり → 409
7. shift.staff_id があれば:
   shiftRepo.findStaffConflictingShifts(
     staffId, newStart, newEnd, officeId, excludeShiftId=shiftId
   )                               → 競合あり → 409
8. shiftRepo.updateShiftSchedule(shiftId, { startTime, endTime, staffId, notes? })
9. auditLogRepo.insert(...)        → success ログ
```

> **既存メソッドの再利用**: `ShiftRepository.findStaffConflictingShifts` / `findClientConflictingShifts` / `updateStaffAssignment` / `updateShiftSchedule` をそのまま呼び出す。  
> `ShiftService.getAdminStaff` は private のため、`AiChatMutationService` が `StaffRepository.findByAuthUserId` を直接呼んで同パターンを実装する。
